# Security Policy

## Supported Versions

The following versions of Carnine are currently receiving security updates:

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please report it by sending an email to **software@carnine.de**. Include as much of the following information as possible:

- Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

This information will help us triage your report more quickly.

## Response Process

1. You will receive an acknowledgement within **48 hours**.
2. We will investigate and provide an estimated resolution timeline within **7 days**.
3. We will notify you when the vulnerability has been fixed.
4. After the fix is released, we will publicly disclose the issue (coordinated disclosure).

## Preferred Languages

We prefer all communications in English or German.

## Scope

This security policy applies to the Carnine backend (Rust) and frontend (Flutter) components, as well as the Protobuf API contract between them.
